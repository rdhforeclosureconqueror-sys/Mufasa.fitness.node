DIRECTORY ARCHITECTURE
GET /api/admin/trainer-directory is protected by requireAuth and admin.trainer_assignments.manage, then by its own per-actor trainer-directory rate limiter. A purpose-built trainerDirectoryService reads users, resolves their configured role, classifies trainer/member candidates, computes assignment status only from active assignment records, deduplicates by stable user ID, and sorts deterministically by display name then user ID. It supports q, type, limit, and offset cursor pagination.

Queries normalize repeated whitespace and trim edges. q is limited to 2–80 characters and rejects control characters. type is restricted to trainer or member. limit is an integer from 1–20. cursor is a non-negative safe integer. Results are capped, stable, and duplicate-free.

DIRECTORY OUTPUT POLICY
The response allowlist is exactly userId, displayName, roleType, accountStatus, and assignmentStatus, plus page-level nextCursor. It does not spread or serialize raw users. Tests seed email, Journey/health, private note, and medical-like fields and verify those values and field names are absent. Admin and super-admin accounts are excluded from member results. Search errors disclose validation policy only, not candidate existence or record contents.

ADMIN AUTOCOMPLETE
Raw IDs are no longer the primary production interaction. The admin page has separate trainer and member comboboxes, a selected-pair review, and a disabled-until-valid assignment action. Stable IDs remain internal after selection while display names are shown.

Requests debounce for 300 ms. Each new search aborts the previous AbortController and uses a monotonically increasing request number plus current-input comparison to ignore stale results. Editing search text clears its old selection. Loading, empty, error/retry, selection, success, and deactivation states are announced through live regions. Duplicate submissions are blocked while a request is active. HTTP status is retained on client errors for status-specific conflict messaging.

The combobox supports Arrow Up/Down, Enter selection, Escape dismissal, mouse/touch selection, aria-activedescendant, aria-expanded, listbox/option roles, and focus return to the input. All result names and history values render with textContent rather than HTML parsing.

DEFECTS FOUND
1. Reproduction: administrator assignment creation required memorizing and typing raw user IDs. Root cause: the Phase 13 admin page exposed only two unassisted text inputs.
2. Reproduction: rapidly changing a future directory search could allow older responses to replace newer results. Root cause: there was no directory search or request lifecycle.
3. Reproduction: trainer program and note fields were identifiable only by placeholder text. Root cause: the inputs lacked explicit labels.
4. Reproduction: narrow layouts and long names could overflow grid tracks or produce controls smaller than a reliable touch target. Root cause: min-content grid sizing and minimal responsive rules.
5. Reproduction: admin errors lost their HTTP status and deactivation could be submitted repeatedly. Root cause: the API helper reduced errors to message-only Error objects and buttons were not disabled during requests.

DEFECTS FIXED
1. Replaced primary raw-ID fields with directory-backed display-name comboboxes and an explicit selected-pair review. Regression: admin autocomplete source/accessibility contract test and directory integration tests.
2. Added 300ms debounce, AbortController cancellation, request sequence checks, and current-input matching. Regression: stale-response protection contract assertions.
3. Added explicit labels to trainer program and private-note controls, plus page/navigation/live-region semantics. Regression: lint and source accessibility contract checks.
4. Added zero-minimum responsive grids, stacking, wrapping, 44px controls, and visible focus styles. Regression: CSS source review and lint.
5. Preserved HTTP status, guarded submitting state, disabled in-flight deactivation buttons, retry feedback, and focused outcome announcements. Regression: duplicate-submission and error-state source contract checks.
